Project Overview

One of the most important products for the Create user are the website themes. For this project I was resposible for reviewing the company's current themes and looking at how they could be evolved. The main goals of this project were:

  • Move away from ‘one design fits all’.
  • A design process to create future themes.
  • Themes that are based on user segments and industry.
  • Manage other design team members to assist with the process.

Skills & Tools

  • Sketching Logo Designs
  • Photoshop
  • Sketch
  • Affinity Designer

The Process

In order to fully understand how our current users were customising the themes and what was offered to them I printed out all of Create's current templates. I used these to create and sort a collection of designs. I segmented them by user type and industry, looking for any commonly occuring patterns.

I assessed different template structures, colour palettes, use of imagery and any connection by industry. I researched the key design elements that can be associated to an indsutry and compared this with how our users were customising the themes to make them their own. The research was presented to the entire company to futher our knowledge of the Create customer base.

The process below is how we planned, designed and built the themes. Each stage of the process was reviewed to ensure the new themes were going in the right direction and meeting the set deadline.

Stage 1

  • Select industry and category for theme. E.g: "Food & Drink" > "Cake Decorating"

  • Create a “template-theme” folder. This was the library for all the fonts, colours, graphics, imagery and notes.

  • Carry our user research based on the chosen industry.

  • Gather and create graphics and imagery for the theme. These were used to create mood boards which and show off the first stage of design in an effective visual way.

  • Pick name to publicly describe the theme. This was used for the website url name.

Stage 2

  • Design theme logo.

  • Create a style board using Sketch, which was made up from the elements below.

  • Add a logo font, which was the same font as the designed logo, and have a fallback font.

  • Heading font, content font and icon font.

  • Create a colour palette, to show how colour logic would work on background and foreground of the theme.

  • Create wallpaper or graphics available for the user upon release.

Stage 3

  • Start buidling the demo site with the use of images, graphics, products and lorem ipsum.

  • Handover to the marketing team to update lorem ipsum with content.

  • Handover theme assets to the dev team. This was then added to the application.

  • Create screenshots and graphics to be used in the theme customisation area of the application and for marketing promotion.

Woollen

About This Theme

Industry - Craft

Industry Focus - Knitting & Craft Material

The visual starting point for Woolen was based around the products, picking out key visual elements to be used in the design. By doing this was able to designed a theme that had light, earthy, warm tones and was soft and playful.

Demo Site - woollen.create.net

Eden

About This Theme

Industry - Home & Garden

Industry Focus - Gardening

Eden's main focus was inspired by old gardening books. The theme needed to feel light, delicate and have mute tones. The colours picked for this came from the plants and pots, which had natural greens and earthy warm orange of the clay pots. I created the logo to feel light and clean but also weathered. Selecting a worn Slab Serif font reflected this well.

Demo Site - eden.create.net

This theme has a very minimal colour palette. Sweeney is a theme that is strong, minimal and luxurious. Using a smaller amount of colours and tones lead to a clean design, using small details such as fine gold lines to do this.
